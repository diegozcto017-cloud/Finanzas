# Instrucciones para Cowork — Proyecto Finanzas Personales

## Contexto
Este es un proyecto Angular 19+ (Standalone, Zoneless) con Tailwind CSS v4 y Supabase como backend.
Todos los archivos fuente están en la carpeta "Finanzas Personales" del escritorio.
El proyecto Angular destino puede ser nuevo o existente.

## Tarea Principal
Lee todos los archivos .ts y .md de la carpeta "Finanzas Personales" y organízalos
en la estructura correcta de un proyecto Angular.

## MODIFICACIONES A ARCHIVOS EXISTENTES

### 1. index.html — Agregar Material Icons
En el archivo src/index.html, dentro del <head>, agregar esta línea si no existe:
```html
<link href="https://fonts.googleapis.com/icon?family=Material+Icons" rel="stylesheet">
```

### 2. package.json — Verificar dependencias
El archivo package.json debe incluir estas dependencias:
```json
{
  "dependencies": {
    "@supabase/supabase-js": "^2.x"
  }
}
```

### 3. Si es proyecto NUEVO (ng new), el app.config.ts debe tener:
```typescript
// src/app/app.config.ts
import { ApplicationConfig, provideExperimentalZonelessChangeDetection } from '@angular/core';
import { provideRouter } from '@angular/router';
import { routes } from './app.routes';

export const appConfig: ApplicationConfig = {
  providers: [
    provideExperimentalZonelessChangeDetection(),
    provideRouter(routes),
  ],
};
```

### 4. Tailwind v4 — Si usa CSS config en vez de JS
Si el proyecto usa Tailwind v4 con @theme en CSS (en vez de tailwind.config.ts),
agregar esto al archivo CSS principal (src/styles.css o src/styles.scss):
```css
@import "tailwindcss";

@theme {
  --color-black-kite: #351E1C;
  --color-garnet: #733635;
  --color-garnet-light: #8B4443;
  --color-garnet-dark: #5C2A29;
  --color-kite-warm: #4A2D2A;
  --color-cream: #F5F0EB;
  --color-warm-gray: #E8E0D8;
}
```

## COMANDOS QUE EL USUARIO DEBE EJECUTAR EN TERMINAL

Después de organizar los archivos, el usuario (Diego) debe ejecutar:

```bash
# 1. Si es proyecto nuevo
ng new finanzas-personales --standalone --style=scss --ssr=false
cd finanzas-personales

# 2. Instalar dependencias
npm install @supabase/supabase-js
npm install -D tailwindcss @tailwindcss/postcss postcss autoprefixer

# 3. Verificar que compila
ng build

# 4. Deploy a Vercel
npx vercel --prod
```

## NOTAS IMPORTANTES

1. Todos los componentes son Standalone (no necesitan módulos NgModule)
2. Todos usan Angular Signals (signal, computed)
3. El template de cada componente está inline (no archivos .html separados)
4. Los estilos usan Tailwind utility classes (no archivos .css/.scss separados)
5. Material Icons se usan via <span class="material-icons">nombre</span>
6. La app es mobile-first con bottom nav en móvil y sidebar en desktop
